# คู่มือการติดตั้งและข้อมูลเฉพาะของ MQL5 Publisher EA

**JornaltradePublisherEA** คือโปรแกรม Expert Advisor ขนาดเล็ก (Lightweight EA) สำหรับติดตั้งบนแพลตฟอร์ม MetaTrader 5 เพื่อดึงข้อมูลประวัติการเทรดและสถานะพอร์ตปัจจุบัน ส่งไปยังระบบเซิร์ฟเวอร์มอนิเตอร์โดยอัตโนมัติ

---

## 🛠️ ขั้นตอนการติดตั้ง (Setup Guide)

### ขั้นตอนที่ 1: การอนุญาตสิทธิ์ WebRequest ใน MT5
ตัว EA จำเป็นต้องส่งข้อมูลผ่านโปรโตคอล HTTP/HTTPS ไปยัง Backend API ซึ่งโปรแกรม MT5 บังคับให้ต้องอนุญาตสิทธิ์ก่อนใช้งานจริง:
1. เปิดโปรแกรม **MetaTrader 5**
2. ไปที่เมนู **Tools** -> **Options** (หรือกด `Ctrl + O`)
3. เลือกแท็บ **Expert Advisors**
4. ติ๊กเครื่องหมายถูกที่ช่อง **"Allow WebRequest for listed URL:"**
5. ดับเบิ้ลคลิกที่รายการว่างด้านล่างเพื่อเพิ่ม URL ของเซิร์ฟเวอร์มอนิเตอร์ของคุณ:
   - หากรันบนเครื่องเดียวกันหรือวง LAN เดียวกัน: `http://127.0.0.1:8088` หรือ IP เครื่องมอนิเตอร์ของคุณ
   - หากใช้เซิร์ฟเวอร์ภายนอก: ใส่โดเมนเนมหรือ IP ของ Cloud VPS ของคุณ
6. กด **OK** เพื่อบันทึกค่า

### ขั้นตอนที่ 2: ติดตั้ง EA ลงใน MT5
1. ดาวน์โหลดหรือคัดลอกไฟล์ [JornaltradePublisherEA.mq5](file:///d:/EA/Thankhun_Tradejornal/mql5/JornaltradePublisherEA.mq5)
2. ในโปรแกรม MT5 ไปที่เมนู **File** -> **Open Data Folder**
3. เปิดเข้าไปยังโฟลเดอร์ `MQL5` -> `Experts`
4. นำไฟล์ `JornaltradePublisherEA.mq5` ไปวางในโฟลเดอร์นี้
5. กลับมาที่โปรแกรม MT5 ในแถบหน้าต่าง **Navigator** ด้านซ้าย ให้คลิกขวาที่หัวข้อ **Experts** แล้วกด **Refresh**
6. จะพบชื่อ `JornaltradePublisherEA` ปรากฏขึ้นมา

### ขั้นตอนที่ 3: เปิดใช้งานและตั้งค่า EA
1. **สำคัญมาก:** เปิดกราฟว่าง (Chart เปล่า) ขึ้นมา 1 กราฟ (เช่น คู่เงิน EURUSD หรือจุดใดก็ได้) **ห้ามรันบนกราฟเดียวกันกับ EA ตัวอื่นที่รันเทรดอยู่** เนื่องจาก MT5 จำกัดให้รัน EA ได้เพียง 1 ตัวต่อ 1 กราฟเท่านั้น
2. ดับเบิ้ลคลิกหรือลาก `JornaltradePublisherEA` ลงบนกราฟว่างนั้น
3. ในหน้าต่างตั้งค่า ให้เลือกแท็บ **Inputs** และกรอกรายละเอียดดังนี้:
   - `InpServerUrl`: ที่อยู่เซิร์ฟเวอร์มอนิเตอร์ของคุณ (เช่น `http://127.0.0.1:8088`)
   - `InpPublisherToken`: รหัส Token สำหรับเชื่อมต่อบัญชีเทรด ซึ่งคัดลอกมาจากหน้าตั้งค่าบัญชีใน Web App (เช่น `JT-XXXXXX...`)
   - `InpSyncInterval`: รอบการส่ง Snapshot สถานะพอร์ต (วินาที) (ค่าเริ่มต้น 60 วินาที)
   - `InpHeartbeatInterval`: รอบการส่งสัญญาณแจ้งว่า EA ออนไลน์อยู่ (วินาที) (ค่าเริ่มต้น 120 วินาที)
4. กด **OK**
5. ตรวจสอบให้มั่นใจว่าปุ่ม **Algo Trading** บนแถบเครื่องมือหลักของ MT5 เปลี่ยนเป็นสีเขียว (เปิดใช้งาน) เรียบร้อยแล้ว

## 🔬 การออกแบบและหลักการทำงานของ EA (Technical Design)

เพื่อหลีกเลี่ยงการดึงพลังเครื่องเทรดหลัก ตัว EA จึงมีโครงสร้างการทำงานแบบ **Event-Driven** และใช้การจำค่าตำแหน่ง (Cursor Persistence):

### 1. ลดโหลดเครื่องแบบ 100% (No Tick Overhead)
* ตัว EA **ไม่ได้ใช้ฟังก์ชัน `OnTick()`** เลย ทำให้การขยับของราคาไม่มีผลต่อความถี่ในการทำงานของ EA
* ใช้ฟังก์ชัน `OnTradeTransaction()` ตรวจจับเฉพาะเมื่อมีการบันทึกธุรกรรมดีลการซื้อขายในบัญชีเท่านั้น เพื่อส่งดีลใหม่ทันที
* ใช้ฟังก์ชัน `OnTimer()` รันทุกๆ 1 วินาทีเพื่อทำหน้าที่ตรวจสอบรอบเวลาการส่งข้อมูลและประมวลผลคำขอที่ค้างคา

### 2. ป้องกันข้อมูลสูญหายและส่งซ้ำ (Cursor Persistence)
* เมื่อรันครั้งแรก EA จะดึงประวัติการเทรดทั้งหมดส่งขึ้นไประบบ (เรียกว่าการ **Bootstrap**) เมื่อส่งสำเร็จจะบันทึกเลขตั๋วดีล (Ticket ID) ล่าสุดเก็บไว้ใน **Global Variables ของ MT5** ภายใต้ชื่อ `JT_LastTicket_<LoginNumber>`
* ในการส่งข้อมูลรอบถัดไป EA จะอ่านเลข Ticket ล่าสุดนี้จากหน่วยความจำ เพื่อนำมาดึงข้อมูลเฉพาะดีลที่เกิดขึ้นใหม่เท่านั้น ทำให้ไฟล์ข้อมูลมีขนาดเล็กมากเพียงไม่กี่ร้อย Bytes
* ค่าใน Global Variables จะคงอยู่แม้ว่าโปรแกรม MT5 จะปิด ปิดคอมพิวเตอร์ หรือเครื่องดับ ทำให้ระบบสามารถมาเชื่อมต่อทำงานต่อได้ทันทีโดยไม่สร้างประวัติการเทรดซ้ำซ้อนในระบบฐานข้อมูล
